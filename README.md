# FAX原稿AI

NEXLINK利用企業向けのFAXDM原稿作成Webアプリです。

サービス内容・ターゲット・問い合わせ導線などを入力すると、AIがFAXDM原稿を生成し、PDFとして出力します。

---

## 1. 概要

- NEXLINK利用企業の担当者が、FAXDM原稿を手早く作成できるWebアプリ
- フォーム入力からAIが以下を自動生成
  - FAXDMの構成案
  - 画像生成用プロンプト
  - A4縦のFAX原稿画像
  - PDF出力用データ
- 生成原稿は一覧で管理し、詳細確認・PDFダウンロード・複製・削除・再生成が可能

---

## 2. 想定ユーザー

- 営業企画担当者
- マーケティング担当者
- 総務・販促担当者
- 中小企業の代表者
- FAXDM原稿作成が負担になる利用者

---

## 3. MVP方針

- ログインあり
- 会社単位でユーザーを管理
- ユーザーは自社作成原稿のみ閲覧可
- 原稿はA4縦のみ
- 出力形式はPDF
- 原稿の直接編集は行わない
- 編集は自然文による再生成指示で対応
- テンプレートはフォーム入力内容の保存として扱う

---

## 4. 主な機能

### 4.1 ログイン

NEXLINK利用企業の担当者がログインして利用します。

### 4.2 原稿一覧

作成済みFAX原稿を一覧表示し、以下を確認できます。

- 原稿タイトル
- サービス名
- ターゲット
- ステータス（下書き / プロンプト生成中 / プロンプト生成完了 / 画像生成中 / 原稿生成完了 / 生成失敗）
- 更新日時
- 詳細表示
- 複製
- 削除

### 4.3 新規作成（2段階生成）

#### 方式

AIによるFAX原稿生成は、以下の2段階で行います。

1. **Step1: プロンプト生成**
   - フォーム入力内容から画像生成用プロンプトを生成する
2. **Step2: 画像生成**
   - ユーザーがプロンプトを確認・編集してから画像生成する

画像生成前にユーザー確認を挟むことで、品質確認とコスト削減を行います。

#### フロー

ログイン → 原稿一覧 → 新規作成 → Step1 基本情報 → Step2 訴求設計 → Step3 掲載情報 → プロンプト生成 → プロンプト確認・編集 → このプロンプトで原稿を生成 → 画像生成 → PDF化 → 原稿詳細

### 4.4 プロンプト確認画面

フォーム入力後に表示される画面で、以下を確認・操作できます。

- 入力内容サマリー表示
- AIによるFAXDM設計サマリー表示
- 画像生成用プロンプトの確認・編集
- プロンプトの再生成
- 「このプロンプトで原稿を生成」による画像生成開始
- 「入力内容に戻る」によるフォーム再編集

### 4.5 原稿詳細

生成済み原稿の詳細画面で以下を行います。

- 原稿プレビュー確認
- PDFダウンロード
- 生成に使用したプロンプトの確認（折りたたみ表示・コピー可）
- プロンプトを編集して再生成
- フォーム内容からプロンプト再生成
- バージョン履歴確認
- 複製
- 削除

### 4.6 再生成

#### A. プロンプトを編集して再生成（MVP優先）

原稿詳細 → プロンプトを編集 → 「このプロンプトで再生成」→ 画像生成 → PDF化 → 新しいバージョンとして保存

#### B. フォーム内容からプロンプトを再生成

原稿詳細 → 「プロンプトを再生成する」→ プロンプト確認 → 画像生成 → PDF化

### 4.7 テンプレート保存

入力済みフォーム内容をテンプレートとして保存し、次回以降の作成時に再利用できます。

テンプレートには image_prompt は保存しません。テンプレートタイトルが未入力の場合は YYYY/MM/DD/hh/mm/ss 形式で自動設定されます。

## 8. Docker / ローカル開発構成

社内プレゼン用MVPとして、最小構成は**web + mysql**のみを想定します。
Redis / Resque / MinIO 等は後付け前提です。

### 8.1 実装済みファイル

- `docker-compose.yaml`
- `Dockerfile.local`
- `config/database.yml`
- `.env.example`
- `.gitignore`

### 8.2 初回起動手順

```bash
docker compose build
docker compose run --rm web bin/rails db:create
docker compose run --rm web bin/rails db:migrate
docker compose up
```

OpenAI連携を使う場合は、`OPENAI_API_KEY` を環境変数に設定してください。設定がない場合は、ローカルの生成ロジックがフォールバックで動作します。

ブラウザで開くURL:

```text
http://localhost:3000
```

### 8.3 この構成でカバーする内容

- ログイン
- 原稿作成フォーム
- AI生成
- 生成画像のローカル保存
- PDF出力
- 原稿一覧
- 原稿詳細
- 編集指示による再生成
- テンプレート保存

生成画像・PDFはまず Rails 標準の `storage/` 配下に保存する想定です。

例:

```text
storage/generated_images
storage/generated_pdfs
```
